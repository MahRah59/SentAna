deployment Checklist

1. Code & Functionality

  • All routes respond correctly (sentiment, emotion, trend, chatbot)
  • No broken links, form errors, or console warnings
  • Transformer and VADER modes switch correctly
  • ABSA and Emotion detection fully functional

2. Input Handling

  • Supports text, file, large text, mock, social media inputs
  • Validations for file types, text length, and required fields
  • Time scale and aspect extraction handled gracefully

3. Model Integration

  • All models load successfully (VADER, Transformers, Emotion)
  • Token limits, chunking, and error handling in place
  • Language detection and translation work as expected

4. Visualization

  • Pie, radar, bar charts display correctly
  • Star-score color scheme is consistent
  • Fallback text appears when data is missing

5. User Interface & UX

  • Mobile responsive layout
  • Clear labels, tooltips, and help texts
  • Animations or transitions enhance user experience
  • Intro video embedded and works properly

6. File & Data Storage

  • No unnecessary saving of uploaded files
  • Database correctly stores sessions, messages, and analysis results
  • Test data cleaned or flagged for deletion

7. Security & Config

  • Password hashing with bcrypt
  • Flask secret key and config variables secured
  • Debug mode disabled
  • Input sanitized for injection or file exploits

8. Deployment Platform

  • Dependencies listed in requirements.txt
  • Deployment scripts (e.g., Procfile, Dockerfile) ready
  • Using Gunicorn or similar WSGI server
  • HTTPS and CORS settings verified

9. Logging & Monitoring

  • TimedRotatingFileHandler set up for daily logs
  • Error logs saved and accessible
  • Optionally, escalation trigger alerts are working
